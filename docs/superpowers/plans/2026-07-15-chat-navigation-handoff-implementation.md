# JobCopilot 聊天页跳转交接实施计划

日期：2026-07-15

关联规格：`docs/superpowers/specs/2026-07-15-chat-navigation-handoff-design.md`

## 任务 1：可测试的聊天导航模块

- [ ] 先增加合法聊天 URL、非 BOSS URL、缺失岗位 ID 和错误岗位 ID 测试。
- [ ] 先增加同标签、子标签、`noopener` 新标签、无关标签和超时测试。
- [ ] 新增 `src/chat-navigation.js`，提供 URL 解析、岗位匹配、导航通道错误识别和一次性观察器。
- [ ] 观察器支持成功、身份失败、超时和主动取消后的完整清理。

## 任务 2：正式投递接入

- [ ] 在后台 Service Worker 中按依赖顺序加载 `chat-navigation.js`。
- [ ] 新增标签页快照与建立聊天页交接函数。
- [ ] 在 `GO_CHAT` 前启动观察器，以聊天页面事实代替内容脚本回执作为成功条件。
- [ ] 对导航造成的消息通道关闭继续等待，对按钮缺失等明确错误立即停止。
- [ ] 只向匹配 `jobId` 的目标聊天页注入脚本并调用 `SEND_BUNDLE`。
- [ ] 增加建联阶段日志，不记录完整 URL、`securityId`、招呼语或密钥。

## 任务 3：标签页安全清理

- [ ] 追踪原详情页、目标聊天页及其是否为当前任务新建。
- [ ] 成功或失败后对临时标签页 ID 去重清理。
- [ ] 不关闭用户原本已打开的搜索、详情或聊天标签页。
- [ ] 保持“三段全部发送成功后才标记已沟通”。

## 任务 4：回归与发布

- [ ] 更新扩展集成测试，确保观察器先于 `GO_CHAT`，匹配聊天页先于 `SEND_BUNDLE`。
- [ ] 更新版本号和 README 故障修复说明。
- [ ] 运行全部测试、JavaScript 语法、Manifest、差异和 API Key 检查。
- [ ] 提交、推送当前分支并更新 PR #2；不执行真实投递。
