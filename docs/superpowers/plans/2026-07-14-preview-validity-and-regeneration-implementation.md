# JobCopilot 预演有效性与重新生成实施计划

日期：2026-07-14

关联规格：`docs/superpowers/specs/2026-07-14-preview-validity-and-regeneration-design.md`

## 任务 1：稳定预演指纹

- [ ] 先扩展 `review-workflow` 测试，证明仅 JD 变化不应过期。
- [ ] 为岗位身份、简历、筛选配置和招呼方案变化增加具体原因测试。
- [ ] 将原始 JD 从精确指纹移除，保留发送前实时资格复核。
- [ ] 迁移现有预演，要求旧指纹安全地重新确认或重新生成。

## 任务 2：单岗位重新生成

- [ ] 先测试重新生成成功、失败保留旧内容和状态回到 `draft`。
- [ ] 增加后台 `REGENERATE_PREVIEW` 消息，只调用模型，不访问 BOSS 和发送路径。
- [ ] 使用已保存完整 JD、当前简历和当前方案重新生成。
- [ ] 持久化更新后的单岗位预演并返回侧边栏。

## 任务 3：确认按钮状态与界面

- [ ] 待确认按钮使用中性灰色可点击样式。
- [ ] 确认期间显示“确认中…”并禁用。
- [ ] 已确认按钮变为绿色“✓ 已确认”并禁用。
- [ ] 编辑开场后恢复待确认状态和正式投递禁用状态。
- [ ] 增加“重新生成 AI 开场”，显示单岗位生成中和错误状态。

## 任务 4：验证与发布

- [ ] 更新版本号和使用文档。
- [ ] 运行全部测试、JavaScript 语法、Manifest、差异和 API Key 检查。
- [ ] 提交并推送当前分支，更新 PR #2。
- [ ] 给出 Chrome 刷新与复测步骤；不执行真实投递。
