## ✅ 任务 1：身份与 JD 测试

- 在岗位身份测试中覆盖：稳定 ID 一致时允许当前公司名或岗位名缺失；明确冲突与 ID 缺失/变化仍阻止。
- 在集成测试中固定预演与正式发送必须检查非空 JD。
- 保留旧的无稳定 ID 身份匹配行为，不把复合 fallback ID 当作稳定 ID。

## ✅ 任务 2：详情与城市清洗

- 在 `job-detail.js` 增加稳定身份判断与“缺失可回填、冲突仍失败”的身份合并规则。
- 在 `background.js` 预演和发送前，对详情结果统一重新应用位置规范化和搜索城市兜底。
- 详情 JD 为空时在进入身份与筛选校验前返回明确错误。
- 增加旧内容脚本污染城市在后台被清洗的测试。

## ✅ 任务 3：图片专属回执

- 将图片 DOM 快照与回执判断提取为可单测模块。
- 图片成功只接受上传后新增的本人消息图片，并要求图片已加载。
- 普通文字气泡增加不得算作图片成功。
- 内容脚本将图片失败转换为 warning；存在成功文字时整单仍成功，纯图片方案仍失败。

## ✅ 任务 4：后台结果与日志

- 将 `completed`、`warnings` 和 `imageConfirmed` 从内容脚本返回后台。
- 根据图片启用和回执结果输出三种准确日志，不再输出“三段式投递成功”。
- 图片 warning 不阻塞下一岗位，核心文字失败继续停止批次。
