# JobCopilot 预演身份容错与图片尽力发送设计

日期：2026-07-15

## 背景

当前预演和正式投递存在三个相关问题：

1. 部分岗位详情页无法解析公司名称，即使稳定岗位 ID 与完整 JD 均可读取，预演仍会因“公司名称缺失”中止。
2. 详情页旧脚本或宽泛 DOM 选择器可能产出类似“验不限硕士瑞德林深圳”的污染城市文本，并被误判为异地。
3. 简历图片发送目前使用“本人消息气泡数量增加”作为成功条件。固定文字气泡延迟出现时，可能被误认成图片发送成功，日志随后错误显示“三段式投递成功”。

用户确认：招呼文字是核心结果，简历图片属于附加内容。图片偶发未发出不能阻塞岗位、整批任务或下一批流程。

## 目标

- 稳定岗位 ID 一致且完整 JD 可读取时，允许公司名称或岗位名称缺失。
- 清除详情页返回的非法城市值；可靠城市缺失时继续使用当前搜索城市“深圳”。
- 将启用的文字消息作为正式投递成功的核心条件。
- 将简历图片改为单次、非阻塞的尽力发送，并准确记录是否得到图片回执。
- 图片未确认时仍标记岗位已沟通、移出当前批次并继续下一岗位。

## 非目标

- 不放宽稳定岗位 ID 校验；ID 缺失或不一致仍立即停止该岗位及当前批次。
- 不在图片失败后自动重试，避免图片延迟到达时重复发送。
- 不保存可续传的部分投递状态，不增加“仅补发图片”入口。
- 不改变预演确认、人工批准或批次失败即停止的其他安全规则。
- 不执行真实 BOSS 投递作为自动测试。

## 一、岗位身份与详情容错

### 1. 核心身份

稳定岗位 ID 是预演与正式发送的强身份锚点：

- 收集岗位和当前详情页都必须有稳定岗位 ID。
- 两个 ID 必须完全一致。
- 当前详情页若明确返回了岗位名称或公司名称，则对应字段仍需与收集记录一致；明确不一致时阻止。
- 当前详情页缺少岗位名称或公司名称时，不再报错，合并结果沿用收集记录中的值。

因此，“缺失”可以容错，“明确冲突”不能容错。

### 2. JD 要求

- 预演必须读取到非空的完整 JD。
- 正式发送前必须再次读取到非空 JD，并继续执行预演指纹与岗位可用性校验。
- 只有详情容器存在但 JD 文本为空时，按“完整 JD 缺失”失败。

## 二、城市数据清洗

详情页返回的数据进入硬筛选之前统一经过当前版本的位置规范化：

1. 只从 `rawLocationFacts` 重新解析已知城市和有效行政区。
2. 不直接信任旧内容脚本返回的任意 `city` 字符串。
3. 可靠页面位置明确为其他城市时，仍按现有规则排除。
4. 没有可靠城市时，使用当前搜索配置中的首个目标城市；本次用户配置即“深圳”。
5. 公司名称、岗位名称、经验或学历拼接形成的污染文本不得进入城市比较。

该规范化同时应用于预演前校验和正式发送前校验，防止刷新扩展后仍在旧标签页运行的内容脚本污染后台状态。

## 三、图片尽力发送

### 1. 核心文字步骤

- AI 个性化开场和固定补充消息继续按启用顺序发送。
- 所有启用的文字步骤必须取得现有文字发送回执；任一文字步骤失败仍立即停止批次。
- 只要至少一个文字步骤成功，岗位就已经建立有效沟通。

### 2. 图片步骤

- 图片上传只执行一次。
- 发送前记录当前本人图片消息集合。
- 上传后等待新的、可见且加载完成的本人图片气泡；普通文字气泡增加不能作为图片成功依据。
- 发现明确上传错误时返回图片警告。
- 等待窗口内没有图片气泡时返回“简历图片未确认”警告。
- 图片警告不改变整单成功结果，不停止批次，也不触发自动重试。

如果当前招呼方案没有启用任何文字、只启用了图片，则图片仍是核心步骤；未确认图片时该岗位失败，避免没有发送任何内容却被标记为已沟通。

## 四、结果与日志

发送脚本返回：

- `success`：核心步骤是否成功。
- `completed`：已完成的步骤。
- `warnings`：非阻塞步骤的警告。
- `imageConfirmed`：图片是否取得专属回执。

后台仅在 `success` 为真时标记岗位已沟通并移出活动批次。

日志分为：

- 图片确认：`✓ 招呼文字和简历图片发送成功`
- 图片未确认：`✓ 招呼文字发送成功｜简历图片未确认，已继续下一岗位`
- 未启用图片：`✓ 招呼文字发送成功`

批次汇总中的“成功”代表核心招呼成功，不再声称所有三段内容均已确认。

## 五、兼容与恢复

- 不迁移旧的岗位进度结构；新增回执字段只存在于本次发送返回值和运行日志中。
- 已成功投递的历史岗位不重新判断图片状态。
- 旧标签页返回的污染城市会在后台校验入口被重新清洗。
- 公司或岗位名称缺失只影响展示信息，不影响由稳定岗位 ID 锚定的发送安全。

## 六、测试与验收

自动测试覆盖：

- 稳定 ID 一致、当前公司名称缺失时通过身份校验。
- 稳定 ID 一致、当前岗位名称缺失时通过身份校验。
- 公司或岗位名称明确变化时仍阻止。
- 稳定 ID 缺失或变化时仍阻止。
- 预演和正式发送都要求非空 JD。
- 污染城市被丢弃并回退到搜索城市深圳。
- 可靠页面城市明确异地时仍排除。
- 延迟文字气泡不能误判为图片成功。
- 新的已加载本人图片气泡才产生图片成功回执。
- 图片上传失败或超时在存在成功文字时只产生警告并继续下一岗位。
- 纯图片方案未确认图片时仍失败。
- 日志不再出现不准确的“三段式投递成功”。

验收执行全部 Node 测试、脚本语法检查、Manifest JSON 检查、`git diff --check` 与 API Key 扫描。浏览器验证仅使用本地 Mock/fixture，不访问 BOSS、不执行真实投递。
