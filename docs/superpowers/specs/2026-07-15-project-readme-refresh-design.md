## 项目定位

JobCopilot 是一个运行在 Edge 和 Chrome 中的 BOSS 直聘 AI 求职工作流扩展。它把岗位收集、完整 JD 读取、硬条件过滤、AI 匹配、人工审批、消息预演、安全投递和岗位进度管理串成一个闭环。

核心原则是“AI 辅助判断，用户负责最终决定”：未批准的岗位不投递，未确认的预演不发送，岗位身份无法精确核验时不发送。

## 功能覆盖范围

README 应覆盖当前整个项目已经具备的能力：

- Xiaomi MiMo、DeepSeek、LongCat 和自定义 OpenAI 兼容接口。
- 多关键词搜索、精准/平衡/宽松模式、关键词扩展和跨搜索去重。
- 可复用筛选方案，以及经验、规模、岗位类型、学历、薪资、城市/行政区、发布时间、必须词、排除词和公司黑名单。
- 完整 JD 读取、可信位置识别、稳定岗位 ID 和发送前复核。
- 快速 AI 判断、按需六维评分、人工补充和人工覆盖。
- 人工批准队列、批量批准、三段式招呼方案、预演编辑和重新生成。
- 正式投递、发送结果回执、批次生命周期、失败岗位重试和岗位进度看板。
- 详情页、模型和聊天页面的有限重试，以及登录、风控、发送中断等全局停止边界。
- 本地存储、预演指纹、图片单份存储和防重复投递。

## 内容原则

- README 是当前项目说明，不是 `1.8.0` 发布说明，也不复制完整提交清单。
- 当前版本号可以作为状态信息展示，但不能主导标题、目录和功能叙事。
- 使用“人工审批、预演确认、身份核验后投递”作为核心定位，不把项目描述成无人值守海投工具。
- 优先使用用户能理解的工作流描述功能；实现模块只在项目结构和可靠性说明中出现。
- 失败处理使用长期有效的行为规则描述，不把某次修复过程写成产品介绍。
- 删除已经不能代表当前界面的旧截图；获得新版截图后再补充。
- 保留正确的仓库克隆地址和扩展加载路径。
- 补充 `node --test tests/*.test.js` 测试命令；测试数量作为当前验证结果说明，不作为长期功能卖点。
