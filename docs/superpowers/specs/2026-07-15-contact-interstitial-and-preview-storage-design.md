# JobCopilot 建联拦截与预演存储去重设计

日期：2026-07-15

## 背景

真实体验已经证明聊天页跳转交接和三段式发送可以成功完成，但仍存在两个会中止批次的问题：

1. BOSS 偶尔在点击沟通后显示“订阅回复消息”二维码弹窗。弹窗覆盖岗位详情页，当前点击器只识别“继续沟通”，后台因此一直等待聊天页并最终超时。
2. 简历图片以 Base64 字符串保存在招呼方案中，同时又被完整复制到每个岗位预演。多次复制会超过 `chrome.storage.local` 默认容量，预演中途出现 `Resource::kQuotaBytes quota exceeded`。

此外，BOSS 聊天路由有时会先进入缺少 `jobId` 的聊天壳，再补齐查询参数。当前观察器把这一瞬时状态立即当成身份校验失败，可能造成误停。

## 目标

- 精确识别并关闭 BOSS“订阅回复消息”弹窗，然后只重试一次沟通动作。
- 聊天页暂时缺少岗位 ID 时提供有限宽限时间，但身份未确认前绝不注入发送脚本。
- 简历图片在每套招呼方案中只保存一份，岗位预演只保存图片指纹。
- 升级时保留已有 AI 开场、预演确认状态和岗位数据，只删除旧预演里重复的图片内容。
- 存储仍失败时显示可理解的中文错误并安全停止，不产生部分投递或自动重试。

## 已选方案

采用用户批准的推荐组合：

1. **限定弹窗识别 + 一次性重试**：只处理同时匹配 BOSS 订阅文案的弹窗，不使用通用“关闭所有遮罩”逻辑。
2. **聊天身份宽限等待**：相关聊天标签先出现无 `jobId` 路由时继续观察最多 5 秒；补齐后严格匹配，超时仍失败。
3. **预演图片引用化**：招呼方案保留图片原文，预演仅保存 `resumeImageFingerprint`；正式发送从同一方案读取图片并核对指纹。

### 方案对比

#### 建联弹窗

- **手动关闭后重新运行**：无需改代码，但批次会中断，无法满足自动投递。
- **精确识别并一次重试（采用）**：兼容已知弹窗，同时限制点击次数和作用范围。
- **自动关闭所有弹窗**：看似通用，但可能误关登录、风险验证或岗位状态提示，不采用。

#### 存储超限

- **增加 `unlimitedStorage` 权限**：可以暂时绕过限制，但保留了每个岗位复制整张图片的错误数据结构，不采用为主方案。
- **图片单份存储、预演保存指纹（采用）**：消除重复数据，并保持预演冻结校验。
- **压缩后继续复制图片**：只能延后超限，图片质量和容量阈值也难以稳定，不采用。

## 组件与边界

### 1. BOSS 建联拦截处理器

`content-search.js` 的沟通点击流程扩展为有限状态机：

1. 定位并点击“立即沟通”或“继续沟通”。
2. 在短时窗口内等待以下页面信号：
   - 出现“继续沟通”；
   - 出现订阅回复弹窗；
   - 页面导航导致内容脚本卸载。
3. 只有弹窗容器同时包含“订阅回复消息”和“使用微信扫码订阅”或等价订阅说明时，才把它认定为目标弹窗。
4. 关闭按钮必须在已确认的弹窗容器内部唯一定位；无法唯一定位时不点击，并返回明确错误。
5. 弹窗关闭并从页面消失后，重新查询沟通入口并最多再点击一次。
6. 第二次仍出现相同弹窗、按钮消失或弹窗无法关闭时立即失败，不循环重试。

处理器不负责判断是否已经进入正确聊天页；最终成功依据仍由后台聊天页观察器提供。

### 2. 聊天页身份宽限

聊天观察器继续以 URL 中与当前岗位完全一致的 `jobId` 作为成功条件。

- 原详情标签或与其有关联的新标签进入聊天路径但暂时没有 `jobId` 时，将该标签标记为“身份待定”。
- 身份待定状态最多保留 5 秒，并且不得超过原有 30 秒建联总截止时间。
- 同一标签随后出现正确 `jobId` 时继续；出现错误 `jobId` 时立即停止。
- 5 秒后仍缺少 `jobId` 时返回“聊天页缺少岗位 ID，已停止发送”。
- 无关且来源不明的聊天标签继续忽略，不能影响当前投递。
- 在身份确认前不得等待聊天脚本、注入 `content-chat.js` 或发送任何消息。

### 3. 预演图片引用模型

招呼方案继续保存：

```text
resumeImageEnabled
resumeImage              // 唯一的 Base64 图片正文
```

岗位预演改为保存：

```text
greetingPlanId
enabledSteps
resumeImageFingerprint   // 图片内容哈希
```

岗位预演不再保存 `resumeImage` 正文。AI 开场、固定消息、完整 JD、稳定输入指纹和确认时间保持现状。

正式投递读取当前批次已校验的招呼方案：

1. 预演的 `greetingPlanId` 必须与当前方案一致。
2. 当前方案完整指纹必须仍与预演输入指纹一致。
3. 若启用简历图片，重新计算当前方案图片哈希，并与 `resumeImageFingerprint` 比较。
4. 三项都通过后，直接把当前方案中的图片正文交给三段式发送器。
5. 任一项不一致都要求重新预演，不使用旧图片、不降级为只发文字。

### 4. 旧数据迁移

加载已有 `sw_previews` 时逐项迁移：

- 保留 `aiOpening`、`fixedMessage`、`jd`、输入指纹、状态和 `confirmedAt`。
- 若旧预演包含 `resumeImage`，优先用已有 `resumeImageFingerprint`；缺失时根据旧图片计算指纹。
- 写回前删除预演中的 `resumeImage` 正文。
- 启用了图片但既没有可用指纹，也无法从旧图片生成指纹时，将该预演标记为过期并要求重新预演。
- 已确认预演在岗位、配置、方案和图片指纹都有效时继续保持已确认。
- 完成招呼方案迁移后删除旧顶层 `resumeImage` 配置副本，避免同一图片在旧字段中继续占用空间。

迁移必须幂等：重复加载不会改变已迁移数据或反复使预演过期。

## 数据流

### 预演

1. 读取岗位详情并完成资格复核。
2. 生成 AI 开场。
3. 创建不含图片正文的预演，保存方案 ID 与图片指纹。
4. 持久化预演状态。
5. 用户检查并确认；确认过程不复制图片。

### 正式投递

1. 重新读取岗位并完成现有全部资格校验。
2. 校验预演状态、稳定输入指纹、方案 ID 和图片指纹。
3. 点击沟通；若出现已知订阅弹窗，精确关闭并只重试一次。
4. 聊天页无岗位 ID 时在安全宽限期内继续观察。
5. 只有精确匹配岗位 ID 后才读取招呼方案图片并执行三段式发送。

## 错误处理与用户提示

- 弹窗无法唯一识别关闭按钮：`BOSS 订阅回复弹窗阻止沟通，请关闭后重试`。
- 关闭后仍再次弹出：`BOSS 订阅回复弹窗重复出现，已停止批次`。
- 聊天页在 5 秒宽限后仍无岗位 ID：`聊天页缺少岗位 ID，已停止发送`。
- 聊天岗位 ID 不一致：继续使用现有身份不一致错误并立即停止。
- 捕获 Chrome 存储配额错误后统一显示：`扩展本地存储空间不足，请重置旧批次或更换较小的简历图片`。

预演持久化失败时：

- 当前岗位标记为预演失败。
- 剩余岗位不继续执行。
- 已成功保存的旧预演保持可见，不自动删除。
- 不允许进入正式投递，也不自动增加存储权限。

## 测试

### 建联处理

- 无弹窗时维持现有点击和导航行为。
- 出现唯一匹配的订阅弹窗时关闭、等待消失并重试一次。
- 不关闭文案不匹配的登录、验证或其他弹窗。
- 找不到唯一关闭按钮时不猜测点击位置并返回明确错误。
- 相同弹窗第二次出现时停止，不形成循环。
- 点击引发页面卸载时仍交由后台观察器确认，不要求内容脚本回执成功。

### 聊天身份

- 相关聊天标签先无 `jobId`、随后补齐正确 ID 时成功。
- 宽限期内补齐错误 ID 时立即失败。
- 5 秒后仍无 ID 时失败。
- 无关聊天标签缺少或包含其他 ID 时不误发。
- 身份确认前不存在 `SEND_BUNDLE`。

### 存储与迁移

- 新预演不包含 `resumeImage` 正文，只包含指纹。
- 多个岗位共用同一方案时，持久化数据只有一份图片正文。
- 正式发送使用方案图片且必须通过图片指纹校验。
- 图片变化后旧预演失效。
- 旧预演迁移后保留开场、确认状态和确认时间，并删除图片正文。
- 缺少图片指纹且无法迁移的预演变为过期。
- 迁移重复执行结果一致。
- 存储配额原始错误转成中文提示。

### 回归验证

- 全部既有测试通过。
- JavaScript 语法、Manifest JSON、差异格式和 API Key 扫描通过。
- 使用 DOM/Chrome API Mock 验收，不访问 BOSS、不执行真实投递。

## 非目标

- 不申请 `unlimitedStorage` 作为主要修复。
- 不压缩、裁剪或改变用户上传的简历图片质量。
- 不自动关闭无法精确识别的 BOSS 弹窗。
- 不放宽岗位 ID 精确匹配规则。
- 不自动循环重试沟通或正式发送。
- 不修改 AI 开场、固定补充消息和岗位筛选逻辑。
- 不以真实投递作为自动化测试。

## 完成标准

JobCopilot 遇到已知订阅回复弹窗时能够安全关闭并完成一次沟通重试；聊天路由短暂无岗位 ID 时可以等待其补齐，但始终只向精确匹配的岗位发送。任意数量的岗位预演都不再为每个岗位复制简历图片；升级后已有有效预演和确认状态得到保留，存储错误提供明确中文提示。
