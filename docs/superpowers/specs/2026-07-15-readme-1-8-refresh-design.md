# JobCopilot 1.8.0 README 重写设计

日期：2026-07-15

## 目标

让 GitHub 首页准确说明 JobCopilot 1.8.0 已从简单的自动投递脚本演进为一套人工可控的 AI 求职工作流，并让首次使用者能够完成安装、配置、审核、预演和投递。

## 文档结构

根目录 `README.md` 是唯一完整文档，按以下顺序组织：

1. 产品定位与适用平台
2. 核心工作流
3. 主要能力与 1.8.0 的可靠性边界
4. 快速开始和实际操作步骤
5. 数据、权限和安全说明
6. 项目结构、测试方式与贡献说明
7. 使用风险、许可证和致谢

扩展目录 `JobCopilot · AI/README.md` 只保留扩展入口、加载路径和根 README 链接，不再维护第二份完整功能文档。

## 内容原则

- 使用“人工审批、预演确认、身份核验后投递”作为核心定位，不把项目描述成无人值守海投工具。
- 使用用户能理解的工作流描述功能；实现模块只在项目结构和可靠性说明中出现。
- 明确 1.8.0 的失败边界：发送前单岗位失败可跳过，登录失效、风控、结果不明确或消息开始发送后的失败停止整批。
- 明确有限重试：详情页持续未就绪重开一次，模型瞬时失败重试一次，聊天页仅接受稳定岗位 ID 精确匹配。
- 补充 `node --test tests/*.test.js` 测试命令和当前 166 项测试基线。
- 不把完整提交清单复制进 README；版本变化只体现在当前能力和可靠性说明中。
- 删除已经不能代表 1.8.0 界面的旧截图，后续获得新版截图后再加入。
- 保留现有安装方式、模型供应商配置、数据本地存储与风险声明。

## 一致性与许可证

- 版本以 `manifest.json` 的 `1.8.0` 为准。
- 模型与默认地址以 `src/llm-client.js` 为准。
- 文件结构以当前 PR 分支实际文件为准。
- MIT License 允许商业使用，因此 README 不再使用与 MIT 冲突的“禁止商业用途”措辞；风险声明聚焦平台规则、自动化风险和使用者责任。
