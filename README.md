## 📖 简介

在 BOSS 直聘海投时，你是否厌倦了：一个个点岗位、复制粘贴千篇一律的招呼语、投到一堆根本不匹配的岗位？

**BOSS 自动投递助手** 帮你把这套重复劳动自动化：

> 设定关键词 → 自动收集岗位 → 读取完整 JD → **AI 按你的简历智能筛选** → 你查看详情并明确批准 → 自动逐个建立联系并发送三段式消息。

每个岗位的第一段开场都基于完整 JD 和你的简历生成，再自动补充固定网站/Agent 消息和简历图片。

## ✨ 功能特性

- 🔍 **多关键词搜索收集**：支持精准、平衡和宽松模式；自动扩展 AI 产品相关词并按稳定岗位 ID 跨搜索去重
- 🎯 **可配置硬筛选**：支持经验、规模、岗位类型、学历、薪资、城市/行政区、发布时间、必须词、排除词和公司黑名单；明确不符会在 AI 调用前排除
- 🤖 **六维 AI 匹配评分**：按岗位方向、工作年限、产品基本功、AI 经历、学历和领域经验生成 0–100 分与证据；支持 Xiaomi MiMo、DeepSeek、LongCat 和自定义接口
- 🔗 **稳定详情读取**：后台通过岗位链接读取完整 JD，不依赖刷新后的搜索卡片
- ✅ **人工批准队列**：查看完整岗位后明确批准；未批准岗位绝不投递
- ✍️ **三段式招呼方案**：AI 个性化开场 → 固定网站/Agent 消息 → 简历图片
- 🧪 **预演确认**：最终消息可查看、可编辑、可重新生成；按钮明确显示待确认、确认中和已确认
- 🔗 **聊天页交接**：兼容订阅回复弹窗和短暂无岗位 ID 的聊天壳；发送前可安全重新建联一次，核对岗位 ID 后才发送
- 💾 **图片单份存储**：简历图片只保存在招呼方案中，预演保存指纹，避免多岗位复制图片导致本地容量超限
- 🛑 **失败立即停止**：任一岗位任一发送步骤失败会停止整个批次
- 🔄 **连续批次处理**：成功岗位自动移入岗位进度，失败和未执行岗位保留；完成后可一键开始下一批
- 🗂️ **岗位进度管理**：已沟通、已回复、面试、淘汰和 Offer 独立管理
- 📊 **实时日志面板**：进度、成功/失败一目了然，支持暂停 / 停止 / 重置
- 🛡️ **拟人化节奏**：随机延迟、逐个投递，自然防风控；已投去重不重复打扰

## 🚀 快速开始

### 1. 下载
```bash
git clone https://github.com/ruiqiyang123/JobCopilot.git
```
或直接 `Code → Download ZIP` 解压。

### 2. 加载扩展（Edge / Chrome 通用）
1. 打开 `edge://extensions`（Chrome 为 `chrome://extensions`）
2. 打开右上角 **开发者模式**
3. 点 **加载解压缩的扩展**，选择仓库中包含 `manifest.json` 的 `JobCopilot · AI` 文件夹
4. 点击扩展图标，打开侧边栏

### 3. 配置
| 配置项 | 说明 |
|--------|------|
| AI 服务商 | 默认 Xiaomi MiMo，也可选择 DeepSeek、LongCat 或自定义 HTTPS OpenAI 兼容接口 |
| API Key | 用于 AI 筛选和生成招呼语；从所选服务商控制台创建 |
| API 地址 / 模型 | 内置服务商自动填充；自定义模式下由用户填写 |
| 简历文字 | 用于完整 JD 筛选和生成 AI 个性化开场 |
| 关键词 / 城市 | 岗位搜索条件 |
| 搜索模式 | 精准只使用填写词；平衡扩展 AI 产品同义词；宽松增加相邻产品方向 |
| 唯一岗位收集上限 | 所有关键词去重后的总岗位数 |
| 硬筛选 | 经验、规模、岗位类型、学历、薪资、行政区、发布时间、必须词、排除词和公司黑名单均可单独配置 |
| 招呼方案 | AI 开场规则、固定补充消息和简历图片；支持多套方案 |

保存前可点击 **测试连接**。测试请求只检查模型接口，不会打开 BOSS 页面或执行投递。

#### Xiaomi MiMo 默认配置

```text
Base URL: https://api.xiaomimimo.com/v1
Model: mimo-v2.5
```

### 4. 使用
**开始收集 + AI 筛选** → 查看完整岗位并 **批准投递** → **预演已批准岗位** → 编辑、重新生成并确认 AI 开场 → **正式投递已批准岗位**。正式投递会安全关闭已识别的订阅回复弹窗，等待并核对实际聊天页；聊天页持续缺少岗位 ID 时，会在发送前自动重新打开并完整复核当前岗位一次，随后再按顺序发送三段内容。文字或图片发送失败时不会自动重试。

投递成功的岗位会立即离开当前审核列表并在岗位进度中标记为“已沟通”；失败和未执行岗位继续保留。批次结束后可处理剩余岗位，或点击 **开始下一批** 清理当前工作区。模型配置、简历、招呼方案、岗位进度和已投防重复记录都会保留。

AI 评分结果分为“推荐、待确认、建议排除”。审核卡片可展开查看六个分项的分数与证据；模型格式异常也会进入待确认。硬筛选已通过的 AI 待确认或建议排除岗位，可由你手动加入候选后再决定是否批准投递。

## 🛠️ 技术栈

- **浏览器扩展**：Manifest V3，原生 JavaScript，无框架
- **AI 模型**：Xiaomi MiMo、DeepSeek 或自定义 OpenAI 兼容模型
- **架构**：Service Worker 编排 + Content Scripts 操作页面 + 侧边栏 UI

## 📁 项目结构

```
src/
├── llm-client.js      # 模型供应商预设、请求、迁移和错误处理
├── job-filters.js     # 多维硬筛选配置、页面事实解析和组合判断
├── job-detail.js      # 稳定岗位链接、身份和详情字段
├── greeting-plans.js  # 多套三段式招呼方案
├── review-workflow.js # 人工审核和预演冻结
├── batch-lifecycle.js # 当前批次投递状态、迁移、过滤和摘要
├── search-strategy.js # 多关键词扩展、搜索模式和跨词去重
├── match-scoring.js   # 六维评分校验、阈值分类和审核排序
├── message-bundle.js  # 三段发送顺序与失败即停
├── chat-navigation.js # 聊天页观察和岗位身份核对
├── contact-retry.js  # 发送前建联重试边界
├── contact-interstitial.js # 订阅回复弹窗安全识别
├── storage-utils.js   # 本地存储配额错误处理
├── background.js      # 收集、完整 JD 筛选、预演和正式投递
├── content-search.js  # 搜索页收集、详情读取和建立沟通
├── content-chat.js    # 三段消息发送与送达确认
├── selectors.js       # DOM 选择器与城市编码
└── sidepanel.*        # 侧边栏界面（配置 / 审核 / 日志）
```

## 🔐 数据与密钥

- API Key 通过插件侧边栏填写，仅保存在浏览器的扩展本地存储中，不写入代码或仓库。
- AI 筛选和招呼语生成会把简历文字、岗位信息及相关 JD 发送给你选择的模型供应商。
- 自定义模型接口仅允许 HTTPS；插件会单独请求该接口域名的访问权限，不会永久申请所有网站权限。
- 审核状态、预演内容和岗位进度保存在浏览器扩展本地存储中；简历图片每套招呼方案只保存一份，岗位预演只保存图片指纹。
