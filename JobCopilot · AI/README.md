## 📖 简介

在 BOSS 直聘海投时，你是否厌倦了：一个个点岗位、复制粘贴千篇一律的招呼语、投到一堆根本不匹配的岗位？

**BOSS 自动投递助手** 帮你把这套重复劳动自动化：

> 设定关键词 → 自动收集岗位 → 读取完整 JD → **AI 按你的简历智能筛选** → 你查看详情并明确批准 → 自动逐个建立联系并发送三段式消息。

每个岗位的第一段开场都基于完整 JD 和你的简历生成，再自动补充固定网站/Agent 消息和简历图片。

## ✨ 功能特性

- 🔍 **多关键词搜索收集**：支持精准、平衡和宽松模式；AI 产品关键词可自动扩展为 Agent、智能体、大模型、LLM、AIGC 等相关搜索并跨词去重
- 🎯 **可配置硬筛选**：经验、规模、岗位类型、学历、薪资、城市/行政区、发布时间、必须词、排除词和公司黑名单均可配置；明确不符会在 AI 调用前排除
- 🤖 **六维 AI 匹配评分**：按岗位方向、工作年限、产品基本功、AI 经历、学历和领域经验生成 0–100 分与证据，并由插件校验和分类
- 🔗 **稳定详情读取**：保存岗位详情链接和稳定 ID，在后台读取完整 JD，不依赖刷新后的搜索卡片
- ✍️ **三段式招呼方案**：AI 个性化开场、固定网站/Agent 消息、简历图片按固定顺序发送
- ✅ **人工批准队列**：推荐、待补充、已批准、不投递、已排除状态分开；未批准岗位绝不投递
- 🧪 **完整预演与冻结**：展示最终三段内容，AI 开场可编辑或单岗位重新生成；按钮明确显示确认状态
- 🔗 **聊天页交接**：兼容订阅回复弹窗和短暂无岗位 ID 的聊天壳，发送前可安全重新建联一次，只向 `jobId` 匹配的聊天页发送
- 💾 **图片单份存储**：简历图片只保存在招呼方案中，预演保存指纹，避免多岗位复制图片导致本地容量超限
- 🛑 **失败立即停止**：详情、身份、建联或任一消息发送异常都会停止整个批次
- 🔄 **连续批次处理**：成功岗位立即移出当前审核队列并进入岗位进度；失败和未执行岗位保留，完成后可一键开始下一批
- 📊 **实时日志面板**：进度、成功/失败一目了然，支持暂停 / 停止 / 重置
- 🗂️ **岗位进度看板**：插件处理过的岗位自动建档，支持已沟通、已回复、面试、淘汰和 Offer 状态
- 🛡️ **拟人化节奏**：随机延迟、逐个投递，自然防风控；已投去重不重复打扰

## 🚀 快速开始

### 1. 下载
```bash
git clone https://github.com/ruiqiyang123/JobCopilot.git
```
或直接 `Code → Download ZIP` 解压。

### 2. 加载扩展（Edge / Chrome 通用）
1. 打开 `edge://extensions`（Chrome 为 `chrome://extensions`）
2. 打开右上角 **开发者模式**
3. 点 **加载解压缩的扩展**，选择仓库中包含 `manifest.json` 的 `JobCopilot · AI` 文件夹
4. 点击扩展图标，打开侧边栏

### 3. 配置
| 配置项 | 说明 |
|--------|------|
| AI 服务商 | 默认 Xiaomi MiMo，也可选择 DeepSeek、LongCat 或自定义 HTTPS OpenAI 兼容接口 |
| API Key | 用于 AI 筛选和生成招呼语；从所选服务商控制台创建 |
| API 地址 / 模型 | 内置服务商自动填充；自定义模式下由用户填写 |
| 简历文字 | 用于完整 JD 筛选和生成 AI 个性化开场 |
| 关键词 / 城市 | 关键词支持逗号、顿号、分号或换行分隔 |
| 搜索模式 | 精准只搜索填写词；平衡扩展 AI 产品同义词；宽松增加智能硬件、AI 语音等相邻方向 |
| 唯一岗位收集上限 | 所有搜索词按稳定岗位 ID 去重后的总数，不是每个词的数量 |
| 工作经验 | 可关闭的分类多选；默认 1 年以内、1–3 年 |
| 公司规模 | 可关闭的分类多选；默认选择所有 100 人以上档位 |
| 更多硬筛选 | 可展开配置岗位类型、学历、月薪、行政区、发布时间、必须词、排除词和公司黑名单 |
| 招呼方案 | AI 开场规则、固定补充消息和简历图片；支持保存多套 |

保存前可点击 **测试连接**。测试请求只检查模型接口，不会打开 BOSS 页面或执行投递。

#### Xiaomi MiMo 默认配置

```text
Base URL: https://api.xiaomimimo.com/v1
Model: mimo-v2.5
```

#### LongCat 默认配置

```text
Base URL: https://api.longcat.chat/openai/v1
Model: LongCat-2.0
```

### 4. 使用

1. 保存基础配置和至少一套完整招呼方案。
2. 点击 **开始收集 + AI 筛选**。插件会按实际搜索词逐轮收集并跨词去重，再读取完整 JD、补全硬筛选事实；明确不符合的岗位不会调用 AI，信息缺失的岗位进入“待补充”。
3. 在“岗位审核”中打开完整岗位，点击 **批准投递** 或 **不投递**。
4. 点击 **预演已批准岗位**，检查并按需编辑 AI 开场；不满意可点击 **重新生成 AI 开场**，满意后点击灰色的 **确认此岗位预演**，成功后按钮会变为绿色 **✓ 已确认**。
5. 点击 **正式投递 N 个已批准岗位** 并确认批次名单。
6. 投递结束后查看成功、失败和未执行摘要。成功岗位会自动移入“岗位进度”；如需继续处理失败或未执行岗位，点击 **处理剩余岗位**；本批结束后点击 **开始下一批**，无需逐个取消批准。

正式投递会再次读取岗位并校验身份、筛选资格和稳定预演配置；遇到已识别的“订阅回复消息”弹窗时会关闭并只重试一次。聊天页短暂缺少 `jobId` 时最多等待 12 秒；若仍缺少，会在发送任何消息前自动重新打开并完整复核当前岗位一次。只有精确匹配岗位后才按“AI 开场 → 固定补充消息 → 简历图片”逐段发送，文字或图片发送失败不会自动重试。任一其他真实失败仍会立即停止，剩余岗位不会自动继续。

审核卡片按综合匹配分排序，可展开查看六个分项证据、主要优势和风险。模型返回非法 JSON、分项越界或总分不一致时，岗位进入“待补充”而不是自动排除。硬筛选已通过的 AI 待确认/建议排除岗位可手动加入候选，人工覆盖记录会随岗位保存。

侧边栏“岗位进度”会自动记录插件收集过的岗位；发送成功后自动标记“已沟通”，并立即从当前审核批次移出。关闭并重新打开侧边栏后，成功岗位不会重新出现在“已批准”中。点击“开始下一批”只清理当前岗位、审核和预演数据，不会删除模型配置、简历、招呼方案、岗位进度或已投防重复记录。第一版不会扫描或导入原有 BOSS 历史。

## 🔄 工作流程

```
┌─────────┐   ┌──────────┐   ┌──────────┐   ┌──────────────────────────┐
│ 配置条件 │ → │ 完整 JD   │ → │ 硬筛选+AI │ → │ 查看详情并明确批准          │
└─────────┘   └──────────┘   └──────────┘   └──────────────────────────┘
                                                          │
                                                          ▼
              对每个选中岗位逐个闭环：
              预演：生成 AI 开场 → 组合固定消息和图片 → 编辑并确认
              正式投递：发送前复检 → 立即沟通 → AI 开场 → 固定消息 → 简历图片
```

## 🛠️ 技术栈

- **浏览器扩展**：Manifest V3，原生 JavaScript，无框架
- **AI 模型**：Xiaomi MiMo、DeepSeek、LongCat 或自定义 OpenAI 兼容模型
- **架构**：Service Worker 编排 + Content Scripts 操作页面 + 侧边栏 UI

## 📁 项目结构

```
src/
├── llm-client.js      # 模型供应商预设、请求、迁移和错误处理
├── job-filters.js     # 多维硬筛选配置、页面事实解析和组合判断
├── job-detail.js      # 稳定岗位链接、身份与详情字段标准化
├── greeting-plans.js  # 多套三段式招呼方案
├── review-workflow.js # 人工审核状态、预演冻结和过期判断
├── batch-lifecycle.js # 当前批次投递状态、迁移、过滤和摘要
├── search-strategy.js # 多关键词解析、扩展模式、公平轮询和跨词去重
├── match-scoring.js   # 六维评分结构校验、阈值分类和审核排序
├── message-bundle.js  # 三段消息顺序与失败即停编排
├── workflow-safety.js # 岗位身份、筛选复检和预演投递安全门
├── chat-navigation.js # 聊天页观察、身份宽限和岗位 ID 核对
├── contact-retry.js  # 发送前建联重试边界
├── contact-interstitial.js # 订阅回复弹窗安全识别
├── storage-utils.js   # 本地存储配额错误处理
├── job-tracker.js     # 本地岗位状态、去重、历史和汇总
├── background.js      # 核心编排：收集→筛选→投递 + 通用模型调用
├── content-search.js  # 搜索页：抓取岗位 + 建立联系
├── content-chat.js    # 聊天页：按三段顺序发送并确认结果
├── selectors.js       # DOM 选择器与城市编码
└── sidepanel.*        # 侧边栏界面（配置 / 审核 / 日志）
```

## 🔐 数据与密钥

- API Key 通过插件侧边栏填写，仅保存在浏览器的扩展本地存储中，不写入代码或仓库。
- AI 筛选和招呼语生成会把简历文字、岗位信息及相关 JD 发送给你选择的模型供应商。
- 自定义模型接口仅允许 HTTPS；插件会单独请求该接口域名的访问权限，不会永久申请所有网站权限。
- 岗位进度和状态历史保存在浏览器扩展本地存储中；简历图片每套招呼方案只保存一份，预演只保存图片指纹。
- 插件不会主动扫描 BOSS 历史会话。
